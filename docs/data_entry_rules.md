# stores.json のデータ入力ルール

## 必須項目
- `id`: 店舗のユニークなID（文字列）
- `name`: 店舗名（文字列）
- `category`: 店舗のカテゴリ（文字列）
- `address`: 店舗の住所（文字列）
- `ward`: 店舗が在る行政区（文字列）
- `lat`: 店舗の緯度（数値）
- `lng`: 店舗の経度（数値）

## 緯度経度の確認方法
- Google Maps などで店舗の正確な位置を確認し、緯度と経度を取得します。
- 緯度は北半球では正の値、南半球では負の値です。
- 経度は東半球では正の値、西半球では負の値です。

## カテゴリ分類ルール
- 店舗の種類を適切に分類します。例：カフェ・喫茶店, スーパー, お好み焼き屋等

## 重複チェック方法
- `id` を確認して既存の店舗と重複がないか確認します。

## 公式サイト確認日 last_checked の記録ルール
- 最終的にデータを更新した日付（YYYY-MM-DD）を記録します。

## 非公式データであることの注意
- 本アプリは非公式です。最新情報は公式サイトをご確認ください。
- 公式サイトの情報を基に追加してください。

## 住所・緯度経度が不明な店舗は追加しないルール
- 店舗の正確な位置情報がない場合は、追加しません。
